MycEvo README Direction

Warm Editorial Flow

Two editable 1600x900 boards exploring MycEvo as research governance: calm field-guide pacing, clear evidence boundaries, and a dominant terracotta validation checkpoint.

Screenshot Targets

  • #product-board explains the evolving workflow mechanism.
  • #technical-board separates public engine, private workspace, and sanitized improvement flow.
  • Native HTML/CSS with inline SVG only; no remote assets or runtime dependencies.

Design-System Notes

Warm editorial pacing is expressed through paper fields, broad margins, restrained outlines, and one terracotta checkpoint per board. The MycEvo brand palette overrides the generic Warm Editorial token values while preserving its field-guide mood.

Strengths

  • Readable at README scale because labels use large SVG text and short line breaks.
  • Validation is visually dominant without implying automation.
  • Public/private architecture and sanitized contribution path are explicit.

Limitations

  • This is an exploratory board, not final production SVG cleanup.
  • Inline SVG keeps editability high, but downstream README export still needs screenshot QA.
  • The composition favors governance clarity over compact mobile viewing.

Semantic Fit

Strong fit for MycEvo: the visual language treats workflow memory as a curated, evidence-gated field guide rather than a magic self-improving loop. Candidate, reusable, validation, and sanitized improvement paths remain semantically separate.
