Structured Research System

MycEvo README Visual Direction

Two editable 1600x900 boards for explaining an evidence-governed self-evolving research workflow harness without implying silent promotion or raw private data upload.

HTML / CSS / Inline SVG

Design-System Notes

Uses the MycEvo light contract: paper field, charcoal type, terracotta for gates, sage and forest for validated or private boundaries. Dark-reference colors are preserved as tokens for later variants, but this README direction stays light for GitHub clarity.

Strengths

The product board makes Validation Gate and no silent promotion visually dominant. The technical board separates external agents, access, public engine, private workspace, evolution gate, and sanitized public improvement into independently scannable zones.

Limitations

This is an exploration, not final production SVG. It avoids icons, logos, and decorative imagery, so the direction depends on structure and typography. At very small README widths, secondary captions should be treated as supporting detail.

Semantic-Fit Assessment

Strong fit for "an auditable research operating system." The disciplined grid, explicit gates, and dashed sanitized path reinforce governance and evidence. Less suited if the README needs a soft marketing hero or broad emotional storytelling.
